# <a name="c26117"></a>C26117
AVERTISSEMENT C26117 : Libération d’un verrou non maintenu \<lock > dans la fonction \<func >.

 La mise en œuvre de paires d' *acquisitions* de verrous d’étendue syntaxiqueC++ et de verrous de libération dans les programmes C/n’est pas effectuée par le langage. Une fonction peut introduire un effet secondaire de verrouillage en effectuant une modification observable de l’état de concurrence. Par exemple, une fonction de wrapper de verrou incrémente le nombre d’acquisitions de verrous, ou nombre de verrous, pour un verrou donné. Vous pouvez annoter une fonction qui a un effet secondaire à partir d’une libération d’acquisition ou de verrouillage de verrou à l’aide de `_Acquires_lock_` ou `_Releases_lock_`, respectivement. Sans ces annotations, une fonction est supposée ne pas modifier le nombre de verrous après qu’elle a été retournée. Si les acquisitions et les mises en production ne sont pas équilibrées, elles sont considérées comme *orphelines*. Un avertissement C26117 est émis lorsqu’une fonction qui n’a pas été annotée avec `_Releases_lock_` libère un verrou qu’elle ne contient pas, car la fonction doit posséder le verrou avant sa libération.

## <a name="example"></a>Exemple
 L’exemple suivant génère l’avertissement C26117, car la fonction `ReleaseUnheldLock` libère un verrou qu’il ne contient pas nécessairement (l’état de `flag` est ambigu) et aucune annotation ne spécifie qu’elle doit.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
        EnterCriticalSection(&p->cs);
    // code ...
    LeaveCriticalSection(&p->cs);
}
```

## <a name="example"></a>Exemple
 Le code suivant résout le problème en garantissant que le verrou libéré est également acquis dans les mêmes conditions.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
    {
        EnterCriticalSection(&p->cs);
        // code ...
        LeaveCriticalSection(&p->cs);
    }
}
```
