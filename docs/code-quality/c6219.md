# <a name="c6219"></a>C6219

avertissement C6219 : cast implicite entre types d’entiers sémantiquement différents : Comparaison de HRESULT à 1 ou TRUE. Envisagez d’utiliser à la place la macro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) ou [failed](/windows/desktop/api/winerror/nf-winerror-failed)

Cet avertissement indique qu’une `HRESULT` est comparée à une valeur explicite non`HRESULT` de un (1). Cette comparaison est susceptible d’entraîner des résultats incorrects, car la valeur de succès type de `HRESULT` (`S_OK`) est 0. Si vous comparez cette valeur à un type booléen, elle est implicitement convertie en false.

## <a name="example"></a>Exemple

Le code suivant génère cet avertissement, car le `CoGetMalloc` retourne un `HRESULT`, qui est ensuite comparé à `TRUE`:

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (hr == TRUE)
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

La plupart du temps, cet avertissement est provoqué par un code qui compare un `HRESULT` à un booléen. Il est préférable d’utiliser les macros `SUCCEEDED` ou `FAILED` pour tester la valeur d’un `HRESULT`. Pour corriger cet avertissement, utilisez le code suivant :

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

Pour cet avertissement, le type de `SCODE` est traité comme un `HRESULT`.

L’utilisation des `malloc` et des `free` (et des API de mémoire dynamique associées) présente de nombreux pièges comme une cause de fuites de mémoire et d’exceptions. Pour éviter ces types de fuites et de problèmes d’exception, utilisez les classes de pointeur et de C++ conteneur fournies par la bibliothèque standard. Il s’agit notamment des [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)et [Vector](/cpp/standard-library/vector). Pour plus d’informations, consultez [pointeurs intelligents](/cpp/cpp/smart-pointers-modern-cpp) et [ C++ bibliothèque standard](/cpp/standard-library/cpp-standard-library-reference).
